# 项目模块文档索引

> 最后更新：2026-06-30

“今日航线”是单用户、本地优先的每日任务工具。当前最真实的核心数据流已经是：晨间输入 → 生成草稿 → 人工确认 → 执行台记录时间段 → 晚间提交与单日复盘 → 需要时复制给 GPT 协作并贴回留档。

## 推荐读取顺序

1. 根目录 `AGENTS.md`：环境锁定、协作流程与验证要求。
2. 根目录 `HANDOFF.md`：当前进度、待办、红线与手机落地状态。
3. 根据任务只读下面对应模块文档，不要无差别全读。

## 模块文档

- `product-usage-guide.md`：面向产品讨论的当前能力说明，不展开技术实现。
- `backend-api.md`：FastAPI 路由、SQLite 模型、执行时间段状态机、Windows / Termux 启动辅助脚本。
- `backend-services.md`：本地规则、兼容旧 AI 入口、GPT 协作文案和安全边界。
- `frontend.md`：今天页、执行台、单日复盘、七日复盘、设置页和 GPT 工作台的前端结构。
- `termux-quickstart.md`：只有手机时，如何在 Termux 里安装、启动和访问本地服务。
- `phone-transfer.md`：如何把当前电脑上的最新项目最省事地传到手机。

## 模块关系

```text
启动今日航线.cmd
  ├─→ app.launcher：等待服务并打开浏览器
  └─→ app.main：启动 FastAPI 与页面路由
          ├─→ app.api：计划、执行、复盘、设置与 GPT 留档接口
          │      ├─→ app.database / app.models：SQLite 与补表
          │      └─→ app.services：本地规则与兼容旧 AI
          └─→ templates + static：今天页、执行台、复盘页与设置页

share-to-phone.ps1
  └─→ dist/daily-plan-termux.zip：给手机下载的临时分发包

Termux
  └─→ termux-start.sh：手机本地启动后端，再由浏览器访问 127.0.0.1
```

## 核心约束

- GPT 只能辅助整理和追问，不能代替用户审批。
- 未完成任务只进入待审池，不自动滚入下一天。
- 执行台同一时刻只允许一个激活段。
- 任务总时间只等于“有效时间 + 计总标签”；中断标签只展示，不并入总时间。
- 敏感原文不进入 AI 请求；密钥不进入网页、数据库或设置接口。
- 当前手机链路只支持“Termux 手动启动后端 + 浏览器访问”，不包含常驻、自启动、离线缓存或 APK 壳。

## 已知实现差异

- 今天页在确认后不再是主要执行入口，主按钮会引导到 `/execute`，主任务实际分钟也改为只读展示。
- 单日复盘页现在除了顶部总统计和晚间收束，还会展示“当天有执行记录的任务看板”。
- 设置页除了原有任务配置，还管理执行标签；GPT 模板编辑仍在 GPT 工作台，不在设置页。
- 七日复盘当前按所选日期所在自然周统计，不是最近 7 个实际执行日。
- `product-usage-guide.md` 已收敛为“当前真实能力说明”，适合直接发给 GPT 作为背景。
